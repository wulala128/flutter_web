# Change: 首页最上方增加轮播图组件

## Why
当前首页顶部没有突出的视觉焦点和活动位，无法在用户进入应用时第一时间展示重点内容（如主推商品、活动 Banner、运营位等），影响电商场景的转化和曝光效率。

## What Changes
- 在首页页面的最上方增加一个轮播图区域，用于展示多张 Banner 图片。
- 支持左右滑动切换轮播项，并在移动端/Web 体验上保持一致的基本交互。
- 支持自动轮播，默认每 3 秒切换一张，可在实现阶段通过配置调整。
- 内置小圆点样式的指示器，持续反映当前展示的 Banner 索引。
- 在视图模型或状态管理中预留轮播数据的统一来源（后续可接真实接口）。
- 为轮播行为定义清晰的要求和场景（首次加载时展示、切换行为、无数据时展示策略等）。

> 注：本提案只定义首页轮播图的产品行为和接口形态，不约束具体 UI 细节（如圆角大小、阴影等），实现阶段可按项目 UI 规范细化。

## Impact
- Affected specs:
  - `specs/home-layout/spec.md`（新增，用于描述首页布局与顶部内容区域）
- Affected code（在实现阶段涉及，不在本提案中修改）:
  - `lib/pages/Home/`：首页布局与内容区域。
  - `lib/viewmodels/home.dart` 或后续的首页数据源管理模块。
